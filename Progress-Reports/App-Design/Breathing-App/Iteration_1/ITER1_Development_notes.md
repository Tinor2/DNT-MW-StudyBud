# ITERATION ONE: Developing the breathing app

- The program is functional, but it has lots of design flaws
    - (expected from an inital prototype)
- Plenty of refinement is required to get to being ready for final results
## CHANGELOG/FEATURES ADDED
    - There are 2 screens: 
        - A beginning screen, which the user sees when they first select the application. The user can select how many cycles of breathing they want to do
        - A screen which guides the user through the breathing process. 
## PMI
### PROS 
    - Firstly, the program is entirely functional
        - The user can control the number of breathing cycles that occur, and begin the program
    - The breathing section works as expected, with some issues with animation
### CONS
    - The sizing of both the begin circle and the cycles circle does not sit right with me at all - the begin button should be MUCH larger, to indicate exactly what the user should be focusing on. In fact, change the shape and form of the cycles button altogether - probably a capsule instead.
    - The label for the cycles button is not visible very well
    - The colours between the blue and the green feel like they're clashing.
    - I should add animation in between selecting the two different buttons
    - It is very difficult to tell which buttons have been added
    - It feels a little to sudden of a change between the beginning screen and the breathing guide screen - maybe add a section in between?
    - Adding a counter to the breathing cycle will make everything a lot clearer
### INTERESTING
    - The breathing section with the cirles that expand and contract to guide the user, feels responsive and a unique way to interact with the user
